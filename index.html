Chapter 1 - History of the markup language

  • HTML is the unifying Language of the World Wide Web and HTML5 is the latest iteration of this lingua franca
  • There was never anything such as HTML 1. The official specifications was HTML 2.0, published by IETF
  • W3C then published XHTML 1.1. It couldn't be served with a mime-type text/html, simply reformulated as XML
  • XHTML 2 was thus introduced. It lacked backwards compatibility with the existing web content or even previous versions of HTML.
  • WHATWG started developing HTML5 as the W3C continued working on XHTML 2
  • HTML5 it's an evolutionary rather than revolutionary in the ongoing story of markup

Chapter 2 - The design on HTML5

  • After the fog of confusion between WHATWG and W3C there was a lot to learn
  • W3C learnt a lesson from post-revolutionary France:changing existing behavior is very, very difficult.

Design Principles

  • HTML5 provides support for previous content
  • Do not reinvent the wheel
  • Pave the cow paths. "If it is not broken, do not fix it"

Error Handling

  • The HTML5 specification doesn’t just declare what browsers should do when they are processing well-formed markup.
  • It also defines what browsers should do when they are dealing with badly formed documents.

Chapter 5 - Semantics

Extensibility

  • Extensibility is about adding more semantic value to elements using the class attribute.
  • New inline elements. This is highlighted.
  • Your profile is 80 80% complete.

Microformats

  • Microformats are a set of conventions which are agreed upon by a community.
  • They are used with classes to extend the HTML5 attributes.
  • However, they are limited by design. They solve 80% of the use cases with 20% of the effort.

Structure

  • HTML5 introduces new structural elements that includes:
    • Section
    • Header
    • Footer
    • Aside
    • Nav
    • Article
  • Caution: Do not replace the new structural element without first thinking of the content.

    • Content Models

  • The new structural elements introduce a completely new content model called “sectioning content.”
